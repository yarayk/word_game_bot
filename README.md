# Words Bot — Отчёт по проекту

## Что это такое

Telegram-бот для групповой игры в слова. Игроки по очереди называют слова на последнюю букву предыдущего, остальные голосуют за каждое слово — настоящее оно или нет. Кто не успел или чьё слово отклонили — выбывает. Побеждает последний оставшийся.

---

## Стек

- **Python 3.11**, aiohttp (чистый, без aiogram)
- **PostgreSQL** + SQLAlchemy async + Alembic
- **Telegram Bot API** — polling вручную

## Классы

**`TgClient`** — отправляет запросы к Telegram. Методы: `get_updates()`, `send_message()`.

**`Poller`** — запускает asyncio task с бесконечным циклом. Получает updates, сдвигает offset, передаёт в handlers. Ошибки не роняют процесс — sleep(1) и продолжает.

**`GameAccessor`** — слой БД. Только SQL-запросы, никакой логики. Методы: `get_active_game`, `create_game`, `update_game`, `get_player`, `get_active_players`, `create_player`, `update_player`, `get_used_words`, `add_used_word`, `get_votes`, `add_vote`, `get_scoreboard`.

**`GameService`** — мозг игры. Только бизнес-логика, никаких SQL-запросов напрямую. Методы: `start_game`, `join_game`, `begin_game`, `submit_word`, `cast_vote`, `resolve_vote`, `eliminate_on_timeout`, `stop_game`.

**`Store`** — собирает все компоненты: `tg_client`, `poller`, `database`, `game` (accessor), `game_service`.

## База данных (4 таблицы)

**`games`** — одна строка на игру. Хранит статус, текущее слово, чей ход, слово на голосовании, дедлайны.

**`players`** — игроки в игре. Хранит очки, активен ли, порядок хода.

**`used_words`** — все слова которые уже были в этой игре. Нужно чтобы не повторяться.

**`votes`** — голоса за каждое слово. Один игрок — один голос за одно слово.

## State Machine

```
IDLE
  ↓ /start_game
WAITING
  ↓ /begin (≥2 игроков)
IN_GAME ←──────────────────────────┐
  ↓ игрок пишет слово              │
VOTING                             │
  ↓ все проголосовали              │
  ├── принято → очки игроку ───────┘
  └── отклонено → игрок выбывает ──┘
  
  Если остался 1 игрок → FINISHED
```

## Команды бота

| Команда | Что делает |
|---|---|
| `/start_game` | Создаёт новую игру в чате, переводит в WAITING |
| `/join` | Игрок вступает в ожидающую игру |
| `/begin` | Запускает игру (нужно минимум 2 игрока) |
| `/stop_game` | Досрочно завершает игру, показывает счёт |
| `/game_info` | Текущий статус игры |
| `+` / `-` | Голосование за слово |

---

## Последовательность игры

1. Кто-то пишет `/start_game` — бот сообщает что игра открыта
2. Игроки пишут `/join` — бот подтверждает каждого
3. Кто-то пишет `/begin` — бот перемешивает порядок, выбирает случайное стартовое слово, объявляет чей ход
4. Игрок чей ход пишет слово на нужную букву
5. Бот проверяет: правильная буква? не использовалось раньше? только русские буквы?
6. Если ok — переходит в VOTING, просит остальных проголосовать
7. Остальные пишут `+` или `-`
8. Когда все проголосовали — бот подводит итог:
   - Больше 50% за → слово принято, очки = длина слова, ход переходит следующему
   - Меньше 50% → слово отклонено, игрок выбывает
9. Если остался 1 игрок — игра завершается, бот показывает победителя и таблицу очков

---

## Игровые правила

- Одна активная игра на чат
- Слово должно начинаться на последнюю букву предыдущего
- Буквы **ь**, **ъ**, **й** в конце слова пропускаются (берётся предпоследняя)
- Слово нельзя повторить в рамках одной игры
- Очки = количество букв в слове
- Автор слова не голосует за своё слово
